The Blackwood Project
The Blackwood Project was our last project at TGA and was an amazing game to work on. It was extremely fun to work on this project all being together in one room and working for 16 weeks on fulltime. The reference game was Mistery, a top down 3D zombie survival murder mystery shooter game. Too many genres for a game.

Lightup Visibility Mechanic

The main mechanic for our horror game was enemies and interactables become visible only in light. To make things more interesting all kinds of lights will make things visible. Point lights, Directional light and Spot lights.

The first thing made was the Flashlight. It used Spotlight and made anything in dark invisible by discarding pixels.
After this we have to only affect Items and enemies by the lights. The line was way too prominent so a gradient of opacity was inserted and it looked a lot like what we wanted for the game.
An issue with this whole thing was that when doing a top down game you want to pretend like the game is 2D because that's the only way you can shoot. So now it's instead a slice of cake that gets lit up and looks way better.

By default this looked not quite right, it's still a hard cut with the blur and the same all the way. What looked better was counting the color strength into the operation and made the switch more random and noisy.

Pointlight and Spotlight shadows

Now that all lights affect monsters if there is something blocking the enemy it will still be lit up. Reason being, all of it is just a math operation checking if the pixel face is looking at the light. Same goes with point lights that need to check for walls.

Spotlight shadows were the simplest being normal Directional shadows but as if it's captured from a camera with a lens.
Point lights are different, they make light all around them. This is accomplished by rendering a shadow map to 6 faces of a cube texture with a 90 degree FOV to capture everything around the light. When rendering the light it checks all 6 sides of the map to see if the pixel is in shadow.

Projected Decals

Before doing projected decals we had mesh planes on the ground as a replacement. It worked for a while until we wanted bullet holes or blood splatters that conformed to the surface hit.

The problem at hand
It's fixed with projected decals, wrapping textures around an object instead of placing a new plane. It was some work but always fun to learn new things. They are made by translating the decal texture the decal cameras view and color it in the main albedo pass.

There is more logic to it but this is put very simply.
The white spheres are the projection cameras that makes the decals
