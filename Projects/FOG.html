FOG
Fog is an upcoming 2 - 4 player horror co-op roadtrip game. I worked on this game most of my internship and it was amazing. In Fog I mainly worked on procedural generation of the world, terrain, road, trees, etc.

Terrain Generation

The terrain generation was not originaly the plan but to make the world the most procedural i had to modify the terrain in the generation. Unreal Engine landscapes can not be modified in runtime, i needed this.

The terrain is created using a given texture with different blue values in it to make height differences.
It was quite the process to get this but put simply it takes the texture and check how big 1 pixel is compared to the terrains width and height. Foreach step I calculated and each pixel, it writes the height from the blue value from the texture to Z height in the game.
The main point of the generated terrain was still to modify it and dig into it to flatten it for buildings and general bumpy ground. Before comiting to creating the mesh i only load the texture so i can modify values after.

I create functions to dig out a circle and a square for what you need it for. After making buildings and roads i go trough each building and at specific increments along the road and flatten it with a circle for roads and squares for buildings.

The following is very exagurated for easier viewing
The buildings were switched dig out circles later in development because it looked nicer.
Using only this the world looks very bleek and needs some color. With a similar technique to the digging out I create a mask Texture to match the coordinates in the same way that can be replaced with realistic dirt in shaders.

It does it the same way as the terrain generation but instead of setting the height we now just set the desigered color for the mask.
What i did not think about was that the texure should be more than per vertex size and could be up to 10x the quality. That is why it lookes blury on edges and you can see the pixels.
At this point the terrain did make the game very laggy because the entire terrain was one object and could not be culled in any way so millions of faces rendered every frame.

Fixing this was a simple procedure, the terrain was split up in multiple mesh components that could all be culled to render way less tris at once.

Road Generation

Before i came in the project had tile based generation where each tile was randomly picked and placed, they had roads build on them. A more powerfull procedurall road generator had to be implemented.

I chose to use parts of Wave Function Collapse algorithm, to not bore you too much i wont be explaining Wave Function Collapse here. But if you are interested here is an explanation of the concept this is a good description, Wave Function Collapse.
I used a mix between complete random generation and Wave Function Collapse. How you make a road is there are small road pieces that you build from unreals spline components. Then i have the rules for WFC on each road piece for what the next pieces can be.

When the generation is done and some post process functions are ran to merge roads we get what the image below shows.
To change what roads was created after one another, an inspector editor for these roads were added and easy enough to set road pieces and the chance for them to place.
This has very primitive road pices and can change but its still very much better than the last generation method. I like what the roads generate very much and im very happy how it turned out.

Next step was creating a visual mesh for the spline. I made this in the unreal engines PCG frame work by going along the road and placing spline meshes at distance increments.
In this iteration of the road generation it was incredibly slow taking 7 seconds for a full generation. I found out about using hlsl shaders to process points to let the gpu do the transformations. It was very fun to learn how to process points using hlsl and remove 90% of the nodes needed before. The generation went from 7 seconds down to less than 0.001 seconds which was insane for how much it still generate.
This was then the finished road with added supports to where it was required on some moutain like terrain.
The last quest was to make the road look more intentional by adding stuff like lightpoles or creating holes in the road.

Best way I think to make lightpoles was splitting the road in pieces and foreach piece go to the left or right and place a lightpole there, add some random offset and the lightpoles looks great.

Bunker building

The most special building i made was bunkers and how the tech works behind the scenes. I did not want to modify the terrain to make the hole down because then it can be used on simple objects like cubes for special occations.

The process was something i had done before but for a whole different reason. We can take the default ground shader and add a global mask that uses the dephtbuffer to remove the pixels covered by the global object mask. We also use a range to check if any player is inside the bunker area and move the object mask to that specific bunker.
The right one is open and you can see through and the left is closed.
The visuals are fixed now so you can see through but if you walk on it you will not fall through because the collision is not changed.

To do this most efficient we remove the terrain layer from player collisions.
We fall through the ground too quick now and the easiest solution is adding some collision buffer like a big rock under the bunker to let the player stand on that instead of the terrain.

We can now easy walk trough the bunker constantly. The view distance of when the floor vissualy disapears is very close at the moment for visuals.

Trees & Foliage

Attempt 1
Generating both trees and foliage is the same process just different amount and remove collisions for grass. The first way i did was just full terrain generation which worked and took not too long to generate.
When tested the game ran terrible, the rason being the collisions of all trees in the map were tested at all times.
I did try to optimise it by only placing trees around the road but if you drove into the firest the trees would just end so would not work
Attempt 2
The next attemt at this was instead of one spawner making all trees i made a spawner that creates tree spawners with a collision box around it. I used the collision box to check if the player should start colliding with the trees in that box or not.
This method helped alot with performance in game but when generating the game it could take up tp a full minute just for the trees to generate once.
Attempt 3
The last and best solution was one they showed as a The Witcher demo for foliage. It used the PCG system but instead of generating all at once it generates and removes foliage around the player localy across all multiplayer clients. The colliders are created along with the meshes as well wich solves the collision system automaticly.
With post processing it looks pretty good i'd say.

Physical Map

the very last element i created was a map to let the players be able to navigate the world. The process was easy by putting multiple scene captures one for each element, one for roads, one for buildings and last for objective.

Unreal has a scene capture component and only the road can render first then only render building icons. If we take all the output textures and add them onto eachother in a material we get a map that we can use.

Town Name Signs

With navigation comes challenges for knowing where you are like a landmark or a sign telling you where you are. They could have been done by just having a basic sign and scaling the text on them to match the default width, Thats boooring.
Thats were i got the idea of proceduraly generated signs using unreal PCG system. I had quite a bit of experience in it with the forest, the road and more so creating it was fun and easy.

The result turned out great and the Text size is what determines the signs final width, the sign can also have varying height for more genuine feel. It also randomly crosses out the town name as if ahead is a ghost town
